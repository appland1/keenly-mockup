🧭 Documented user flow + what's actually built today

Source: The flow below is documented in Hennie's Figma file → "Mobile and Web_4 Sina" page → "User Flow" diagram at the top of the canvas. It's the intended product design, not something we invented. View it directly: open in Figma (requires read access on appland@pm.me).

What Hennie's Figma "User Flow" diagram describes

📤 Share + manage
  • Story → Share → Modal → Invite user → Notification
  • Story → Copy link (public link)
  • Story → Delete → Confirmation modal → Delete or Cancel
🖥 Ingress: Website
  • App Store / Google Play badges visible on landing
  • OR mobile-web version (no install)
  • First-time: Welcome → Get Started → Pick pre-made OR Create new → Sign-in
  • Repeat: Sign-in → My Stories → Continue or Create new
📱 Ingress: Mobile app
  • First-time user: Create new OR Pre-made stories → Selected story
  • Repeat user: Create new / Pre-made / My Stories → Selected story

What's actually built on livekeenly.com today

None of the flow above is implemented on the desktop website yet. The only steps that work today are Sign In and Sign Up. After login, the user lands back on the homepage with "Hi, [Name]" in the nav — there is no dashboard, no story library, no story creator, no share, no anything. You literally can't post anything or do anything on the site beyond authenticating.

🔁 Ongoing maintenance (after launch)

Building it is the easy part. Keeping it healthy is the long game. Here's the realistic monthly engineering cadence after launch.

Task Frequency Hours / month What it covers
SEO + speed audit Monthly ~1-2 hrs Google Search Console scan · Lighthouse re-run · sitemap re-submission if new pages · Core Web Vitals · indexing issues · keyword ranking shifts.
Firebase / dependency security patches Monthly ~1 hr npm audit · Firebase SDK updates · Vite + React minor versions · any critical CVE patches.
Error log review (Sentry) Weekly ~30 min/wk = ~2 hrs/mo Review new errors users hit · triage + fix or backlog · alert escalation for critical bugs.
Uptime + cost monitoring Weekly ~15 min/wk = ~1 hr/mo UptimeRobot dashboard · Firebase Blaze billing check · Hostinger usage check · alert if cost spike.
Content moderation (via CMS) Weekly or as flagged Hennie's job — ~1-2 hrs/mo Once stories go public, Hennie reviews flagged content via CMS. Not engineering — but real time.
Backup verification Monthly ~30 min Test that Firestore export + Firebase backups are working. Verify we can restore.
User support tickets As they arrive Varies (0-10+ hrs/mo) Reset passwords, refund requests, account merges, lost data, "how do I..." questions. Scales with user count.
Total monthly engineering minimum — ~5-8 hrs/month Plus reactive support + new feature requests on top.
Why this matters: SaaS apps aren't "build once, walk away." Firebase pushes SDK updates monthly. Google changes how it indexes pages. Browsers update. Users find edge cases. Without ongoing maintenance, the site degrades quietly — slower load times, missing search results, expired SSL certs, broken integrations. ~5-8 engineering hours/month keeps it healthy. Compare that to the cost of a full rebuild every 18 months when neglect catches up.
